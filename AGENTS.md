# AGENTS.md

本文件为 AI Agent 及项目开发者提供工作约定。

---

## 关键决定记录

在 Agent 工作过程中或开发者提示词中，当出现以下情况时，视为**关键决定**，需要记录：

- 选定了一种架构方案而排除了其他方案
- 确定了会影响后续开发的命名、目录结构、数据格式
- 对现有设计做出了重大变更
- 确定了技术选型或依赖
- 解决了存在争议或两难的设计问题

### 记录位置

所有关键决定文档放在 `docs/decisions/` 目录中。

### 文件命名

```
docs/decisions/000-决定摘要.md
```

- 编号从 `000` 开始递增，使用三位数字
- 摘要使用简短的中文描述，以连字符分隔

### 文档模板

每个决定文档至少包含以下内容：

```markdown
# 000 — 决定标题

**日期**: YYYY-MM-DD

**状态**: 提议 | 已通过 | 已废弃

## 背景

描述做出此决定时的上下文和问题。

## 决定

明确陈述做出的决定。

## 原因

解释为什么选择这个方案而不是其他候选方案。

## 影响

描述此决定对项目的影响：
- 正面影响
- 负面影响或代价
- 后续需要跟进的事项

## 备选方案

列出曾考虑过的其他方案及其被拒绝的原因。
```

### 版本

若无关键决定，不需要强行创建文档。
